# AI-034 - Deepfake Tool Execution/Access

> **Detection ID:** AI-034
> **Status:** Active
> **Risk Score:** 50 (user) + 30 (system)
> **MITRE ATT&CK:** T1588.005 - Obtain Capabilities: Exploits (Resource Development)
> **Data Model:** Endpoint.Processes
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Staging
> **Last Updated:** 2026-02-22

---

## Overview

This detection identifies the execution of known deepfake and synthetic media tools on corporate endpoints. It matches process names against the `ai_deepfake_tools.csv` lookup, which catalogs deepfake tools across five categories: face swap, voice cloning, lip sync, avatar generation, and image generation. Each tool is assigned a risk level (critical, high, or medium) based on its potential for misuse.

Deepfake technology poses a distinct class of threat to organizations. Face-swap tools (DeepFaceLab, FaceFusion, Roop) can create convincing video impersonations of executives for business email compromise or wire fraud. Voice cloning tools (RVC, ElevenLabs, TortoiseTTS) can replicate anyone's voice from a few seconds of audio, enabling vishing attacks that bypass voice-based authentication. These tools have been used in real-world attacks where fraudsters impersonated CEOs to authorize wire transfers exceeding $200,000.

The detection fires when any process matching the lookup executes on an endpoint, regardless of the user's role. The system risk score (30) is intentionally higher than most detections because the presence of deepfake tooling on a corporate endpoint is inherently concerning -- these tools are rarely needed for legitimate business purposes outside of specific creative and security teams.

The lookup-driven approach means new tools can be added without modifying the detection search. The `enabled` field in the lookup allows specific tools to be disabled if they generate excessive false positives (e.g., Stable Diffusion in design-heavy organizations).

## Detection Logic

### Plain English

1. Query the `Endpoint.Processes` data model for all process execution events, collecting the process name, process path, full command line, user, and destination host.
2. Convert the process name to lowercase and look it up against `ai_deepfake_tools.csv` to identify known deepfake tools.
3. Filter to only enabled tools (where `tool_enabled=1` in the lookup).
4. Aggregate results by user, source host, and process name, collecting the tool name, category, risk level, and execution count.
5. Exclude sanctioned users/systems via `ai_unsanctioned_filter`.
6. Tag the event with `detection_id=AI-034`.

### Key Fields

| Field | Description | Example Value |
|---|---|---|
| `user` | User who executed the tool | `mwilson@corp.com` |
| `src` | Endpoint where tool was executed | `DESKTOP-A1B2C3` |
| `process_name` | Executable name matched | `deepfacelab.exe` |
| `tools` | Human-readable tool name(s) | `DeepFaceLab` |
| `categories` | Tool categories matched | `face_swap` |
| `risk_levels` | Risk level from lookup | `critical` |
| `executions` | Number of execution events | `5` |
| `process_path` | Full path to executable | `C:\Users\mwilson\deepfake\deepfacelab.exe` |
| `process` | Full command line | `deepfacelab.exe --gpu 0 --target face.mp4` |

### Tool Categories

| Category | Tools | Risk Implication |
|---|---|---|
| `face_swap` | DeepFaceLab, FaceFusion, Roop, DeepFaceLive, FaceSwap, SimSwap | Video impersonation for fraud or social engineering |
| `voice_clone` | RVC, ElevenLabs, TortoiseTTS, XTTS, Bark | Voice impersonation for vishing or auth bypass |
| `lip_sync` | Wav2Lip | Realistic talking-head video manipulation |
| `avatar_generation` | SadTalker | Animated avatar creation from still images |
| `image_generation` | Stable Diffusion (webui.py), ComfyUI | Synthetic image creation (lower risk alone) |

### Thresholds

| Parameter | Default Value | Notes |
|---|---|---|
| Execution count | 1+ | Any single execution triggers the detection |
| Suppress period | 1 hour | Suppressed per user/src/process_name combination |

## Triage Steps

> **Estimated triage time:** 15-20 minutes

### Step 1: Validate the alert and identify the tool

```spl
index=risk search_name="AI RBA - AI-034*" risk_object="$user$" earliest=-24h
| table _time risk_object risk_score risk_message
| sort -_time
```

- Note the specific tool name, category, and risk level from the risk message.
- Critical-risk face swap and voice clone tools require immediate investigation.
- Medium-risk image generation tools may require less urgency.

### Step 2: Identify the user and their role

```spl
| inputlookup identity_lookup_expanded
| search identity="$user$"
| table identity first_name last_name email managedBy category priority bunit
```

- Users in creative, marketing, or video production teams may have legitimate need for some of these tools.
- Security teams may use deepfake tools for detection testing or awareness training.
- Users in finance, HR, legal, or executive roles running face-swap or voice-clone tools are highly suspicious.

### Step 3: Examine the process execution details

```spl
| tstats summariesonly=t count values(Processes.process) as process values(Processes.process_path) as process_path values(Processes.parent_process_name) as parent_process from datamodel=Endpoint.Processes where Processes.user="$user$" Processes.dest="$src$" Processes.process_name="$process_name$" earliest=-24h by Processes.user Processes.dest Processes.process_name _time span=1h
| rename Processes.* as *
| table _time user dest process_name process process_path parent_process count
| sort -_time
```

- **Process path:** Is the tool installed in a standard location (Program Files) or a temp/download directory? Temp installations suggest the user is trying to avoid detection.
- **Parent process:** Was the tool launched from Explorer (manual), cmd/PowerShell (scripted), or a browser download (just downloaded)?
- **Command-line arguments:** Look for target file references, GPU flags, or output paths that reveal intent.

### Step 4: Check what files the tool accessed

```spl
| tstats summariesonly=t count values(Filesystem.file_name) as files values(Filesystem.file_path) as file_paths from datamodel=Endpoint.Filesystem where Filesystem.user="$user$" Filesystem.dest="$src$" earliest=-4h by Filesystem.action _time span=1h
| rename Filesystem.* as *
| search file_name="*.mp4" OR file_name="*.avi" OR file_name="*.wav" OR file_name="*.mp3" OR file_name="*.jpg" OR file_name="*.png"
| table _time action files file_paths count
| sort -_time
```

- Video and audio file access around the time of tool execution indicates active deepfake creation.
- Look for files containing executive names, customer names, or other identity-relevant filenames.
- Check if the user accessed corporate headshot directories or voice recording systems before the tool execution.

### Step 5: Check if any deepfake output was distributed

```spl
| tstats summariesonly=t count values(Email.subject) as subjects from datamodel=Email where Email.src_user="$user$" earliest=-24h by Email.src_user Email.recipient _time span=1h
| rename Email.* as *
| where count>0
| table _time src_user recipient subjects count
| sort -_time
```

- Check if the user sent emails with attachments after the deepfake tool execution.
- Look for uploads to file sharing platforms (SharePoint, Google Drive, Slack) around the same time.

### Step 6: Check for related endpoint activity

```spl
| tstats summariesonly=t count values(Processes.process_name) as processes from datamodel=Endpoint.Processes where Processes.user="$user$" Processes.dest="$src$" earliest=-4h by _time span=15m
| rename Processes.* as *
| table _time processes count
| sort -_time
```

- Look for video/audio editing tools (ffmpeg, audacity, premiere) running alongside the deepfake tool.
- Look for screen recording or webcam capture tools that might be feeding the deepfake tool.
- Look for download managers or torrent clients that were used to obtain the deepfake tool.

### Step 7: Correlate with other risk events

```spl
index=risk risk_object="$user$" earliest=-24h
| stats sum(risk_score) as total_risk values(search_name) as detections by risk_object
```

- AI-034 + AI-036 (compromised plugin) = user may be installing a broader set of malicious AI tools.
- AI-034 + AI-015 (after-hours) = deepfake creation during off-hours suggests evasion.
- AI-034 + AI-RISK-005 (multi-vector) = deepfake tool is part of a broader attack campaign.

### Step 8: Make a determination

- **True Positive:** Employee running face-swap or voice-clone tools with no legitimate business need, especially if targeting specific individuals.
- **Benign True Positive:** Creative/marketing team using approved video production tools, or security team testing deepfake detection capabilities.
- **False Positive:** Process name collision with a non-deepfake tool (very rare given the specificity of the lookup).

---

## True Positive Indicators

- Tool category is `face_swap` or `voice_clone` (highest risk categories).
- User is not in a creative, marketing, or security team.
- Process path indicates a temp or hidden directory installation.
- Command-line arguments reference specific individuals (executives, customers).
- The user accessed corporate headshot or voice recording repositories before running the tool.
- Deepfake output files were subsequently uploaded, emailed, or shared.
- The tool was executed after hours or on a weekend.
- Multiple deepfake tools from different categories were executed in the same session.
- The parent process is a script or automation tool rather than manual launch.

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Creative/marketing team using video production tools for approved content | High | Allowlist creative teams for specific tool categories (image_generation, avatar_generation) |
| Security team testing deepfake detection capabilities or creating awareness training materials | Medium | Allowlist security team with time-bounded entries |
| Research or academic use in R&D departments exploring synthetic media | Medium | Require documented approval; add to sanctioned entities with expiration |
| Stable Diffusion/ComfyUI used for legitimate design work (non-face-related) | Medium | Consider disabling medium-risk image_generation tools in the lookup if they generate excessive noise |
| Process name collision with unrelated software | Low | The lookup is specific to known deepfake tool binaries; collisions are rare |

## Response Actions

### If confirmed True Positive

1. **Contain:** Immediately isolate the endpoint if active deepfake creation targeting specific individuals is confirmed. Disable the user's network access to prevent distribution of deepfake content.
2. **Assess targets:** Determine whose identity was being impersonated. If executives, customers, or partners were targeted, notify them immediately so they can warn their contacts.
3. **Collect evidence:** Preserve the deepfake tool installation, input files (source photos/videos/audio), and any generated output. These are critical for determining the scope of the threat.
4. **Check distribution:** Search email, file sharing, and messaging systems for any deepfake content that was already distributed. If found, work with platform administrators to remove it.
5. **Assess impact:** Determine if any deepfake content was used in social engineering, fraud, or impersonation attempts. Check for unauthorized wire transfers, account changes, or access requests that coincide with the timeline.
6. **Escalate:** Notify legal counsel (deepfake creation may have legal implications), HR, and the CISO. For fraud or impersonation cases, involve law enforcement.
7. **Remediate:** Remove the deepfake tools from the endpoint. Implement application control policies to prevent future installation.

### If Benign True Positive

1. Add the user or team to `ai_sanctioned_entities.csv` with the specific tool category and an expiration date.
2. Ensure the team has documented approval for using deepfake tools.
3. Recommend using isolated/sandboxed environments for deepfake-related work to avoid triggering endpoint detections.
4. Close the notable as "Benign True Positive."

## Tuning Guidance

### Reducing false positives

- If your organization has creative teams that routinely use image generation tools, consider setting `enabled=0` for `image_generation` category tools in `ai_deepfake_tools.csv`.
- Allowlist specific teams rather than individual users for easier maintenance.
- Consider separate suppression policies for `critical` vs. `medium` risk tools.

### Reviewing the deepfake tool lookup

```spl
| inputlookup ai_deepfake_tools.csv
| table tool_name process_name category risk_level enabled notes
| sort risk_level category
```

Review the lookup periodically to add newly released deepfake tools and remove deprecated ones. The synthetic media landscape evolves rapidly.

### Adding new tools to the lookup

When a new deepfake tool is identified, add it to `lookups/ai_deepfake_tools.csv`:
```csv
tool_name,process_name,category,risk_level,enabled,notes
NewTool,newtool.exe,face_swap,critical,1,Description of the new tool
```

### Allowlist entries for creative teams

```
entity_type,entity,provider,scope,expires,owner,reason,enabled,allow_key
group,creative-team,Deepfake Tool,group,2026-12-31,creative-director,approved_video_production,1,creative-team|Deepfake Tool
```
