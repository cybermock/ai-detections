# AI-005 - Unsanctioned AI Desktop App Execution

> **Detection ID:** AI-005
> **Status:** Active
> **Risk Score:** 20 (user) + 12 (system) = 32 total
> **MITRE ATT&CK:** T1204.002 - User Execution: Malicious File (Execution)
> **Data Model:** Endpoint.Processes
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Installation
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies execution of AI desktop applications including ChatGPT, Claude, Cursor, Windsurf, Pieces, LM Studio, Jan, Continue, Tabnine, Trae, Zed, and Warp. Desktop AI applications can operate outside the browser, making them harder to monitor through traditional web proxy controls. They may also cache data locally and maintain persistent connections to AI backends.

## Detection Logic

### Plain English

1. Query `Endpoint.Processes` for all process creation events.
2. Enrich process names and original file names against `ai_tool_processes.csv` using two lookup passes (by process_name_lc and original_file_name_lc).
3. Apply fallback matching by process_path for macOS `.app` bundle names.
4. Filter to processes where `usage_type="desktop_app"` or matching `.app` path patterns.
5. Filter where `enabled=1` in the lookup.
6. Exclude sanctioned entities via `ai_unsanctioned_filter`.

### Key Fields

| Field | Description | Example Value |
|---|---|---|
| `user` | User who launched the app | `jsmith` |
| `src` | Endpoint | `LAPTOP-42` |
| `process_name` | Desktop app process name | `cursor` |
| `process_path` | Full path to the application | `/Applications/Cursor.app/Contents/MacOS/cursor` |
| `provider` | AI provider | `Cursor` |

## Triage Steps

> **Estimated triage time:** 5-10 minutes

1. **Validate the alert:** Confirm the process name and user are real.
2. **Identify the user:** Check their role. Developers using AI-assisted IDEs may be common.
3. **Review the application:**
   ```spl
   | tstats summariesonly=t values(Processes.process_path) as app_path values(Processes.original_file_name) as original_file
     from datamodel=Endpoint.Processes where Processes.user="$user$" Processes.dest="$src$" Processes.process_name="$process_name$" earliest=-24h
   | table app_path original_file
   ```
4. **Check software inventory:** Verify if the application is on the approved software list.
5. **Check for related network activity:** Look for AI-019 (code assistant network connections).

---

## True Positive Indicators

- Application is not on the approved software list.
- User installed the application without IT approval.
- The application is communicating with unsanctioned AI backends (see AI-019).
- Application is running on a production server or sensitive endpoint.

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Developer using an approved AI IDE (Cursor, Windsurf) | High | Allowlist development teams at the group level |
| Process name collision (e.g., `jan` could be a different app) | Low | Validate via process_path and original_file_name |
| AI feature in a general-purpose app (e.g., Zed has AI optional) | Medium | Consider if the AI feature is active or dormant |

---

## Response Actions

### If confirmed True Positive

1. Notify the user and manager about the unapproved application.
2. Request IT to remove the application from the endpoint.
3. Review what data the application may have accessed or sent.
4. Document the finding.

### If Benign True Positive

1. Add the user/group to `ai_sanctioned_entities.csv` with the specific provider.
