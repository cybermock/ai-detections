# AI-046: MCP Server Execution

## Overview
| Field | Value |
|-------|-------|
| **Detection ID** | AI-046 |
| **MITRE ATT&CK** | T1059 — Command and Scripting Interpreter |
| **Kill Chain Phase** | Execution |
| **Risk Score** | User: 35, System: 20 |
| **Data Model** | Endpoint.Processes |
| **Cron** | */5 * * * * |

## Description
Detects execution of Model Context Protocol (MCP) servers that extend AI assistant capabilities beyond chat to include file system access, database queries, API calls, browser control, and infrastructure management. MCP servers are launched via Node.js (npx), Python (uvx), or other runtimes and categorized by type: filesystem, github, database, messaging, browser, search, infrastructure, and memory.

## Alert Rationale
MCP servers dramatically expand the attack surface of AI assistants by granting them access to local resources. A filesystem MCP server allows an AI tool to read/write any file the user can access. A database MCP server grants query access. A GitHub MCP server can create PRs and modify repositories. This represents a significant data access and integrity risk that most organizations have not addressed in their AI governance policies.

## Triage Steps

### 1. Identify the MCP Server Type and Scope
```spl
| tstats count values(Processes.process) as commands
  from datamodel=Endpoint.Processes
  where Processes.dest="$src$" Processes.process="*mcp*"
  by Processes.user Processes.process_name _time span=5m
| sort _time
```

### 2. Classify Risk by MCP Server Type
| MCP Type | Risk Level | Concern |
|----------|-----------|---------|
| filesystem | **Critical** | Full file read/write access |
| database | **Critical** | Direct database query access |
| github | **High** | Repository access and modification |
| infrastructure | **High** | Docker/K8s management access |
| browser | **High** | Web scraping and automation |
| messaging | **Medium** | Slack/Discord message access |
| search | **Low** | Web search capabilities |
| memory | **Low** | AI context persistence |

### 3. Check What AI Tool Is Using the MCP Server
```spl
| tstats values(Processes.parent_process_name) as parent
  from datamodel=Endpoint.Processes
  where Processes.dest="$src$" Processes.process="*mcp*"
  by Processes.user Processes.process_name
```

### 4. Assess Data Access Scope
- What directories/databases/repos does the MCP server have access to?
- Is the MCP server configuration restricted to specific paths?
- Are there any sensitive files or databases in scope?

## Known False Positives
- Developers using MCP servers as part of their authorized AI-assisted development workflow
- CI/CD pipelines using MCP servers for automated testing or deployment
- Authorized AI integrations that use MCP for structured data access
- MCP servers running in sandboxed/containerized environments with limited scope

## Tuning Recommendations
- Add sanctioned entity entries for authorized MCP server users and systems
- Weight risk scores by MCP type (filesystem/database higher, search/memory lower)
- Exclude known development machines or CI/CD systems
- Consider reducing cron frequency if MCP usage is authorized and widespread

## Escalation Criteria
- **Critical**: Filesystem or database MCP server on production infrastructure
- **High**: GitHub or infrastructure MCP server on non-development endpoints
- **Medium**: Browser or messaging MCP server on corporate endpoints
- **Low**: Search or memory MCP server in development environments

## Response Actions
1. Identify which AI tool is connected to the MCP server and what resources it accesses
2. If unauthorized, terminate the MCP server process immediately
3. Audit file access, database queries, or API calls made through the MCP server
4. Review the MCP server configuration for overly broad permissions
5. Check if any sensitive data was accessed or exfiltrated via the AI tool
6. Update AI governance policy to include MCP server usage guidelines
7. Consider implementing network segmentation to restrict MCP server outbound connections
