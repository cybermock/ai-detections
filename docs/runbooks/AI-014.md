# AI-014 - Local LLM Listening Port Detected

> **Detection ID:** AI-014
> **Status:** Active
> **Risk Score:** 30 (system only)
> **MITRE ATT&CK:** T1571 - Non-Standard Port (Command and Control)
> **Data Model:** Network_Traffic
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Installation
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies active TCP connections to well-known local LLM service ports on localhost (127.0.0.1 or 0.0.0.0). The monitored ports are: Ollama (11434), LM Studio (1234, 43411), LocalAI/llama.cpp (8080), Jan.ai (1337), and GPT4All (4891). Active connections to these ports indicate a running LLM inference server, which represents unauthorized local AI infrastructure.

## Detection Logic

### Plain English

1. Query `Network_Traffic` for TCP connections to specific local LLM ports (11434, 1234, 43411, 8080, 1337, 4891).
2. Filter to localhost traffic (source or destination is 127.0.0.1 or 0.0.0.0).
3. Map the destination port to a known LLM service name.
4. Aggregate by source: count connections, list services and ports.
5. Exclude sanctioned entities.

### Key Fields

| Field | Description | Example Value |
|---|---|---|
| `src` | System with the LLM server | `10.1.2.50` |
| `llm_services` | Identified LLM services | `Ollama` |
| `ports` | Active LLM ports | `11434` |
| `connections` | Number of connections observed | `42` |

## Triage Steps

> **Estimated triage time:** 5-10 minutes

1. **Identify the system and its owner:**
   ```spl
   | inputlookup asset_lookup_by_str | search ip="$src$"
   | table ip dns nt_host category owner bunit
   ```
2. **Check for AI-011 (local LLM process execution)** on the same system.
3. **Determine which LLM service is running** based on the port.
4. **Check if this is a known development or research machine.**
5. **Note:** Port 8080 is commonly used by non-AI services (web servers, proxies). Correlate with AI-011 to confirm LLM usage.

---

## True Positive Indicators

- Port is specific to LLM (11434 = Ollama, 43411 = LM Studio) rather than generic (8080).
- AI-011 (LLM process execution) also fired on the same system.
- AI-013 (model download) preceded this detection.
- System is not classified as a development/research machine.

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Port 8080 used by a web server, proxy, or other non-AI application | High | Correlate with AI-011 to confirm LLM usage; consider removing 8080 from the search |
| Developer running authorized local LLM for testing | Medium | Allowlist the system |
| Port scan or ephemeral connection that happens to hit a monitored port | Low | Check connection count; single connections may be noise |

---

## Response Actions

### If confirmed True Positive

1. Identify the LLM software and models running on the system.
2. Determine if the system is accessible from the network (0.0.0.0 binding is exposed to all interfaces).
3. Notify the system owner and IT.
4. If the LLM server is network-accessible, contain immediately (firewall rule or service shutdown).

### If Benign True Positive

1. Add the system to the allowlist for "Local LLM" provider.
