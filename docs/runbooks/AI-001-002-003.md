## Deprecation Notice

**AI-001, AI-002, and AI-003 have been deprecated and consolidated into AI-028 (Unsanctioned AI Web Access - Consolidated).**

### Why were these detections deprecated?

The original design created individual detections for each AI web provider (ChatGPT, Claude, Gemini). This approach had several limitations:

1. **Maintenance burden:** Each new AI provider required a new saved search stanza, leading to configuration sprawl.
2. **Inconsistent coverage:** Only three providers were covered while 50+ AI web domains exist.
3. **Duplicate logic:** All three searches were structurally identical, differing only in the `Web.url_domain` filter.
4. **Risk inflation:** A user visiting both ChatGPT and Claude would trigger two separate detections (50 total risk points) for what is effectively the same behavior category.

### What replaced them?

**AI-028** is a single consolidated detection that:
- Covers all AI web domains defined in `ai_provider_domains.csv` (50+ domains).
- Uses the `ai_domains_filter` macro for domain-to-provider resolution.
- Provides identical risk scoring (15 user + 10 system = 25 total).
- Fires once per user/provider combination rather than once per detection per provider.

### Migration steps

1. Disable AI-001, AI-002, and AI-003 in `savedsearches.conf` by setting `disabled = 1`.
2. Enable AI-028 (which is active by default in the updated configuration).
3. Update any dashboards, reports, or saved searches that reference AI-001, AI-002, or AI-003 to reference AI-028 instead.
4. Risk index data from AI-001/002/003 will age out naturally over the retention period.

### What if I still need the individual detections?

If your SOC workflow requires per-provider detection stanzas (e.g., for different response procedures per provider), keep AI-001/002/003 active and disable AI-028. However, the recommended approach is to use AI-028 with provider-specific response actions based on the `provider` field in the notable.

## Original Detection Logic (for reference)

Each detection queried the `Web` data model for traffic to a single AI provider's web domain:
- AI-001: `chatgpt.com`, `chat.openai.com` (OpenAI)
- AI-002: `claude.ai` (Anthropic)
- AI-003: `gemini.google.com` (Google Gemini)

The searches enriched via `ai_domains_filter`, filtered via `ai_unsanctioned_filter`, and assigned 15+10 risk.
