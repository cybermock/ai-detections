# AI-011 - Local LLM Framework Execution

> **Detection ID:** AI-011
> **Status:** Active
> **Risk Score:** 20 (user) + 15 (system) = 35 total
> **MITRE ATT&CK:** T1204.002 - User Execution: Malicious File (Execution)
> **Data Model:** Endpoint.Processes
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Installation
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies execution of local LLM frameworks including Ollama, llama.cpp (llama-server, llama-run, llama-cli), LM Studio, llamafile, Jan, GPT4All, KoboldCpp, LocalAI, vLLM, MLX, Open WebUI, and text-generation-launcher. Local LLMs can process sensitive data entirely offline, bypassing network-based monitoring. While this avoids cloud data exposure, it introduces risks around unauthorized software installation, GPU resource consumption, and uncontrolled model usage.

## Detection Logic

### Plain English

1. Query `Endpoint.Processes` for processes matching known local LLM tool names.
2. Enrich via `ai_process_filter` macro; filter to `usage_type="local_llm"`.
3. Set provider to "Local LLM" as default.
4. Exclude sanctioned entities.

---

## Triage Steps

> **Estimated triage time:** 5-10 minutes

1. **Validate the process name and user.**
2. **Check the endpoint context:**
   ```spl
   | tstats summariesonly=t values(Processes.process) as command values(Processes.process_path) as path
     from datamodel=Endpoint.Processes where Processes.user="$user$" Processes.dest="$src$" Processes.process_name="$process_name$" earliest=-24h
   | table command path
   ```
3. **Determine if the endpoint has a GPU** (local LLM usage is common on GPU workstations).
4. **Check for AI-014 (listening port)** -- indicates the LLM server is actively running and accepting connections.
5. **Check for AI-013 (model download)** -- indicates the user recently downloaded model files.

---

## True Positive Indicators

- User is not in a role that justifies running local AI models.
- The endpoint is a corporate laptop without GPU (unusual for legitimate LLM work).
- LLM framework was installed without IT approval.
- Activity is accompanied by model downloads (AI-013).

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Data scientist or ML engineer running approved local models | High | Allowlist the user/group for "Local LLM" provider |
| Developer evaluating local LLM tools as part of product development | Medium | Allowlist with expiration |
| IT team testing endpoint AI capabilities | Low | Allowlist IT service accounts |

---

## Response Actions

### If confirmed True Positive

1. Determine what models the user has downloaded and is running.
2. Verify what data is being processed through the local LLM.
3. Notify the user's manager and IT about unauthorized software.
4. Request removal of the LLM framework if not approved.

### If Benign True Positive

1. Add to the allowlist with "Local LLM" as the provider.
