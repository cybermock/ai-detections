# AI-021 - AI Browser Extension Activity

> **Detection ID:** AI-021
> **Status:** Active
> **Risk Score:** 12 (user) + 8 (system) = 20 total
> **MITRE ATT&CK:** T1176 - Browser Extensions (Persistence)
> **Data Model:** Web
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Installation / Persistence
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies the installation or active usage of AI browser extensions by monitoring web traffic to known AI extension domains and extension store pages. AI browser extensions (ChatGPT, Copilot, Claude, Perplexity extensions) can read page content, intercept form data, and send information to AI services without the user explicitly navigating to an AI website. This makes them a stealthy vector for data exposure.

## Detection Logic

### Plain English

1. Query the `Web` data model for traffic to known AI extension backend domains (extensions.chat.openai.com, browser.claude.ai) or extension store pages (Chrome Web Store, Mozilla Add-ons) with AI-related keywords.
2. Aggregate by user/source: count activity, collect extension URLs and user agents.
3. Exclude sanctioned entities.

### Key Fields

| Field | Description | Example Value |
|---|---|---|
| `user` | User with the extension | `jsmith@corp.com` |
| `src` | Endpoint | `10.1.2.50` |
| `extension_activity` | Number of extension-related requests | `15` |
| `extension_urls` | URLs indicating extension activity | `extensions.chat.openai.com/...` |
| `user_agents` | Browser user agent | `Mozilla/5.0 ...Chrome...` |

---

## Triage Steps

> **Estimated triage time:** 5 minutes

1. **Determine if the traffic indicates installation (store pages) or active usage (extension backend domains).**
2. **Check which extension is being used:**
   ```spl
   | tstats summariesonly=t values(Web.url) as urls from datamodel=Web
     where (Web.url_domain IN ("extensions.chat.openai.com","browser.claude.ai") OR Web.url="*chrome.google.com/webstore*chatgpt*") Web.user="$user$" earliest=-24h
   | table urls
   ```
3. **Review the organization's browser extension policy.** Are AI extensions explicitly prohibited?
4. **Check if the extension is collecting page data** by looking for POST requests to AI extension backends.

---

## True Positive Indicators

- Extension is actively communicating with AI backends (not just installed).
- Organization has a policy prohibiting AI browser extensions.
- User handles sensitive data that the extension could intercept.
- Extension was recently installed (store page access followed by backend traffic).

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| User browsing the Chrome Web Store and viewing AI extension pages without installing | Medium | Check for subsequent backend domain traffic to confirm installation |
| IT-approved browser extension for a sanctioned AI service | Medium | Allowlist users with approved extensions |
| Ad or tracking script loading content from a domain that matches the pattern | Low | Verify URLs are genuine extension endpoints |

---

## Response Actions

### If confirmed True Positive

1. Notify the user about the policy violation.
2. Request IT to remove the extension via browser management policy (Chrome Enterprise, Firefox Enterprise).
3. Consider deploying a browser extension allowlist to prevent future installations.

### If Benign True Positive

1. Add to the allowlist if the extension is approved.
