> **Data Model:** Network_Traffic
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Actions on Objectives
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies IDE and code editor processes (VS Code, Cursor, Windsurf, JetBrains IDEs, Visual Studio) making network connections to AI backend services (OpenAI, Anthropic, GitHub Copilot, Codeium, Sourcegraph, etc.). This correlates the application with the AI destination, confirming that code editors are actively communicating with AI services. The user field is typically "unknown" as network traffic data often lacks user attribution.

## Detection Logic

### Plain English

1. Query `Network_Traffic` for connections where the application is a known IDE/editor AND the destination matches a known AI backend domain.
2. Map destinations to AI providers.
3. Aggregate by source: count connections, list IDE apps and AI backends.
4. Exclude sanctioned entities.

### Key Fields

| Field | Description | Example Value |
|---|---|---|
| `src` | Endpoint running the IDE | `10.1.2.50` |
| `app` | IDE application name | `code` (VS Code) |
| `dest` | AI backend domain | `api.githubcopilot.com` |
| `provider` | AI provider | `Microsoft Copilot` |
| `connections` | Number of connections | `85` |

---

## Triage Steps

> **Estimated triage time:** 5 minutes

1. **Identify the system and its owner:**
   ```spl
   | inputlookup asset_lookup_by_str | search ip="$src$"
   | table ip dns nt_host owner category bunit
   ```
2. **Determine which IDE and AI backend are communicating.**
3. **Check if the AI extension/plugin is approved** (e.g., GitHub Copilot may be sanctioned while other plugins are not).
4. **Look for AI-005 (desktop app execution)** to correlate with process-level detection.

---

## True Positive Indicators

- IDE is connecting to an unsanctioned AI backend.
- The endpoint owner is not a developer.
- Multiple AI backends are being contacted from the same IDE.

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Developer using sanctioned GitHub Copilot in VS Code | High | Allowlist development endpoints for approved IDE AI services |
| IDE extension auto-updating or checking for updates via AI domain | Low | Check connection frequency; infrequent connections may be updates |
| Network proxy/CDN resolving to an AI domain | Low | Verify the destination is an actual AI API endpoint |

---

## Response Actions

### If confirmed True Positive

1. Determine which IDE extension is making the AI connections.
2. Notify the user and request removal of unsanctioned extensions.
3. Consider blocking AI API domains at the proxy for non-approved endpoints.

### If Benign True Positive

1. Allowlist the system for the specific AI provider.
