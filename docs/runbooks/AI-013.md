# AI-013 - LLM Model File Download

> **Detection ID:** AI-013
> **Status:** Active
> **Risk Score:** 30 (user) + 18 (system) = 48 total
> **MITRE ATT&CK:** T1105 - Ingress Tool Transfer (Command and Control)
> **Data Model:** Web
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Installation
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies downloads of LLM model files (GGUF, GGML, SafeTensors formats) from model repositories such as HuggingFace, Ollama registry, and ModelScope. Large model downloads (default threshold: 500 MB or 3+ files) indicate that a user is setting up local LLM infrastructure, which can bypass cloud-based monitoring and DLP controls.

## Detection Logic

### Plain English

1. Query the `Web` data model for traffic to model hosting domains (huggingface.co, registry.ollama.ai, ollama.com, modelscope.cn) or URLs containing model file extensions (.gguf, .ggml, .safetensors).
2. Filter to URLs containing model-related path patterns (resolve, blob, library, /v2/, download, thebloke).
3. Calculate download size in MB.
4. Aggregate by user and source: count downloads, sum total MB.
5. Load the model download threshold from `ai_detection_config.csv` (default: 500 MB).
6. Filter to users exceeding the MB threshold or downloading 3+ files.
7. Exclude sanctioned entities.

### Thresholds

| Parameter | Default Value | Config Lookup Key |
|---|---|---|
| Download size threshold | 500 MB | `model_download_mb` |
| File count threshold | 3 downloads | N/A (hardcoded) |

## Triage Steps

> **Estimated triage time:** 5-10 minutes

1. **Review what models were downloaded:**
   ```spl
   | tstats summariesonly=t values(Web.url) as urls sum(Web.bytes_in) as bytes_in
     from datamodel=Web where Web.user="$user$" Web.url_domain IN ("huggingface.co","registry.ollama.ai","ollama.com") earliest=-24h
   | eval mb_in=round(bytes_in/1024/1024,2)
   | table urls mb_in
   ```
2. **Identify the user and their role.** ML engineers and data scientists may legitimately download models.
3. **Check for AI-011 (local LLM execution)** to see if the models are being actively used.
4. **Determine the model type.** General-purpose chat models vs. specialized models for code generation.
5. **Check total download volume.** Multi-GB downloads suggest large models (7B+ parameters).

---

## True Positive Indicators

- User is not in ML/AI/data science role.
- Download volume is multi-GB (large models).
- Downloads are followed by AI-011 (local LLM execution) and AI-014 (listening port).
- Models are from providers with data handling concerns (e.g., modelscope.cn).

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| ML engineer downloading models for approved research | High | Allowlist ML teams for "Local LLM" and "Model Registry" providers |
| Automated CI/CD pulling model artifacts for testing | Medium | Allowlist build systems |
| Developer browsing HuggingFace documentation (small downloads) | Low | The 500 MB / 3-file threshold should filter out browsing |

---

## Response Actions

### If confirmed True Positive

1. Determine what models were downloaded and their intended use.
2. Verify if the user has approval for local LLM infrastructure.
3. Notify the user's manager and IT.
4. If models are from untrusted sources, scan for embedded malware (model files can contain executable code).

### If Benign True Positive

1. Add to the allowlist with "Local LLM" or "Model Registry" provider.
