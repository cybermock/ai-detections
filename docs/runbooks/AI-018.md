# AI-018 - Multiple AI Services In Single Session

> **Detection ID:** AI-018
> **Status:** Active
> **Risk Score:** 15 (user) + 8 (system) = 23 total
> **MITRE ATT&CK:** T1567.002 - Exfiltration Over Web Service: Exfiltration to Cloud Storage (Exfiltration)
> **Data Model:** Web
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Actions on Objectives
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies users who access 3 or more distinct AI service providers within a single hour. Accessing multiple AI services in quick succession may indicate "prompt shopping" (trying the same sensitive query across multiple providers), data distribution across multiple exfiltration channels, or comparison testing of AI services. The default threshold is 3 distinct providers per hour.

## Detection Logic

### Plain English

1. Query the `Web` data model for AI domain traffic, bucketed by 1-hour spans.
2. Enrich domains against `ai_provider_domains.csv`.
3. Count distinct providers per user/source per hour using `eventstats`.
4. Load the multi-service threshold from `ai_detection_config.csv` (default: 3).
5. Filter to users accessing >= 3 distinct providers in one hour.
6. Exclude sanctioned entities.

### Thresholds

| Parameter | Default Value | Config Lookup Key |
|---|---|---|
| Multi-service threshold | 3 providers / hour | `multi_service_threshold` |

## Triage Steps

> **Estimated triage time:** 5 minutes

1. **Review which providers were accessed:**
   ```spl
   index=risk search_name="*Multiple AI*" risk_object="$user$" earliest=-24h
   | table _time risk_object risk_score
   ```
2. **Determine if the user was comparing services or sending the same data to multiple providers.**
3. **Check for AI-007 (high upload) across any of the providers.**
4. **Consider the context:** A developer evaluating AI tools is less concerning than an HR employee accessing 4 AI services.

---

## True Positive Indicators

- User uploads data to multiple providers (same content to different services).
- User is not in a role that requires AI tool evaluation.
- Activity is combined with high upload volumes.
- User accessed both web interfaces and APIs.

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Developer comparing AI coding assistants | High | Allowlist developers; increase threshold to 4-5 |
| User has browser tabs open to multiple AI services (background traffic) | Medium | Verify actual interaction vs. passive page loads |
| Security researcher evaluating AI services | Medium | Allowlist with specific scope |

---

## Response Actions

### If confirmed True Positive

1. Determine if the same data was submitted to multiple providers.
2. Notify the user's manager about multi-service usage.
3. If data shopping is confirmed, escalate to data governance.

### If Benign True Positive

1. Add to the allowlist if multi-service usage is justified.
