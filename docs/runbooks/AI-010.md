# AI-010 - Repeated Unsanctioned AI Access Burst

> **Detection ID:** AI-010
> **Status:** Active
> **Risk Score:** 18 (user) + 10 (system) = 28 total
> **MITRE ATT&CK:** T1071.001 - Application Layer Protocol: Web Protocols (Command and Control)
> **Data Model:** Web
> **Schedule:** `*/5 * * * *` (every 5 minutes)
> **Lookback Window:** `-15m@m` to `now`
> **Kill Chain Phase:** Actions on Objectives
> **Last Updated:** 2026-02-13

---

## Overview

This detection identifies users making a high number of requests to unsanctioned AI providers within a 15-minute window. The default threshold is 10 requests. Burst activity can indicate automated tools, rapid data pasting, or programmatic API usage that warrants investigation. On its own the risk is moderate (28 points), but it frequently fires alongside other detections to build a risk picture.

## Detection Logic

### Plain English

1. Query the `Web` data model for all traffic to AI domains in the past 15 minutes.
2. Enrich via `ai_domains_filter`.
3. Aggregate request counts by user/source/provider.
4. Load the `request_burst_count` threshold from `ai_detection_config.csv` (default: 10).
5. Filter to users exceeding the threshold.
6. Exclude sanctioned entities.

### Thresholds

| Parameter | Default Value | Config Lookup Key |
|---|---|---|
| Request burst count | 10 requests / 15 min | `request_burst_count` |

---

## Triage Steps

> **Estimated triage time:** 5 minutes

1. **Validate the request count and user.**
2. **Determine if the volume is normal for the user's role:**
   ```spl
   | tstats summariesonly=t count from datamodel=Web where Web.user="$user$" earliest=-7d by _time span=15m
   | stats avg(count) as avg_requests perc95(count) as p95_requests max(count) as max_requests
   ```
3. **Check user agent:** Browser traffic vs. programmatic access (curl, python-requests).
4. **Check for AI-008 (scripted CLI) and AI-007 (high upload) co-occurrence.**
5. **Determine if the user is actively working or if this is automated.**

---

## True Positive Indicators

- Request count significantly exceeds the threshold (e.g., 50+ in 15 minutes).
- Requests are programmatic (non-browser user agent).
- Activity occurs outside business hours.
- User is not a developer.

---

## False Positive Scenarios

| Scenario | Likelihood | Mitigation |
|---|---|---|
| Normal web browsing with many subrequests (CSS, JS, API calls) | High | Increase threshold to 20-30 based on environment |
| Developer actively coding with AI assistant generating many API calls | High | Allowlist developers for their approved provider |
| Real-time chat interface generating frequent XHR/fetch requests | Medium | Adjust threshold based on observed chat patterns |

---

## Response Actions

### If confirmed True Positive

1. Review what data is being sent in the requests.
2. Notify the user's manager if the activity is unsanctioned.
3. Consider rate-limiting or blocking the user's access at the proxy.

### If Benign True Positive

1. Add to the allowlist or increase the burst threshold.
