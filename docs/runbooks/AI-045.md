# AI-045: Voice Cloning API Access

## Overview
| Field | Value |
|-------|-------|
| **Detection ID** | AI-045 |
| **MITRE ATT&CK** | T1588.005 — Obtain Capabilities: Exploits |
| **Kill Chain Phase** | Staging |
| **Risk Score** | User: 40, System: 20 |
| **Data Model** | Web |
| **Cron** | */5 * * * * |

## Description
Detects web and API access to known voice cloning and speech synthesis services including ElevenLabs, PlayHT, Resemble AI, WellSaid Labs, Murf AI, LOVO AI, Deepgram, Speechify, Coqui, FakeYou, Uberduck, and Voicemod. Voice cloning technology can be weaponized for social engineering attacks, executive impersonation, vishing (voice phishing), and deepfake audio fraud.

## Alert Rationale
Voice cloning services have been used in documented attacks to impersonate executives in wire fraud schemes, bypass voice-based authentication systems, and create convincing social engineering attacks. Access to these services from corporate endpoints warrants investigation, especially from non-creative departments.

## Triage Steps

### 1. Identify the User and Department
```spl
index=risk search_name="AI RBA - AI-045*" earliest=-24h
| stats count sum(risk_score) as total_risk values(provider) as services values(src) as systems by risk_object
| lookup identity_lookup_expanded identity as risk_object OUTPUT department priority
```

### 2. Review Access Patterns
```spl
| tstats count sum(Web.bytes_out) as bytes_out values(Web.url) as urls
  from datamodel=Web
  where Web.url_domain IN ("api.elevenlabs.io","elevenlabs.io","play.ht","resemble.ai","murf.ai","fakeyou.com")
  Web.user="$user$"
  by Web.url_domain _time span=1h
```

### 3. Determine Business Justification
- Is the user in a department with legitimate voice synthesis needs? (Marketing, media production, accessibility, customer experience)
- Is this API access (indicating programmatic use) or web access (indicating exploration/personal use)?
- What volume of data is being uploaded? (Audio samples for cloning indicate active cloning activity)

### 4. Check for Related Activity
```spl
index=risk search_name="AI RBA*" risk_object="$user$" earliest=-7d
| search search_name IN ("*phishing*","*email*","*social*")
| stats count by search_name
```

## Known False Positives
- Marketing and media production teams using voice synthesis for legitimate content creation
- Accessibility teams using text-to-speech services
- Customer experience teams building voice interfaces or IVR systems
- Security teams conducting authorized red team exercises

## Tuning Recommendations
- Create sanctioned entity entries for users/departments with legitimate voice synthesis needs
- Differentiate between web browsing (low risk) and API access with audio uploads (higher risk)
- Adjust risk score based on department sensitivity
- Consider disabling this detection if your organization actively uses voice synthesis tools

## Escalation Criteria
- **Critical**: API access with significant data upload (>10 MB) suggesting active voice cloning
- **High**: Access from executive or finance department (potential impersonation staging)
- **Medium**: Repeated access from non-creative department
- **Low**: Single web visit to a voice cloning service

## Response Actions
1. Determine if the user has a legitimate business need for voice cloning
2. If unauthorized, investigate what audio samples were uploaded and whose voice was being cloned
3. Alert potentially impersonated individuals (especially executives)
4. Review recent phishing or vishing reports that may correlate with voice cloning activity
5. If malicious intent confirmed, preserve evidence and escalate to incident response
6. Update AI acceptable use policy to explicitly address voice cloning services
