Correlation Engine

RBA Correlation Rules

Five correlation searches that aggregate risk events into actionable notables.

AI-RISK-001
Medium Risk Threshold
User risk score reaches or exceeds 60 points accumulated within a 24-hour sliding window. Generates a medium-severity notable event for analyst triage.
user risk ≥ 60pts → medium notable
AI-RISK-002
Critical Risk Threshold
User risk score reaches or exceeds 90 points accumulated within a 24-hour sliding window. Generates a critical-severity notable event requiring immediate response.
user risk ≥ 90pts → critical notable
AI-RISK-003
Data Collection to AI Upload Kill Chain
Detects the pattern of data collection events (file access, clipboard, database queries) followed by AI service upload within a 4-hour window. Indicates potential staged exfiltration.
60 pts + notable
AI-RISK-004
Privilege Escalation Plus AI Usage
Correlates privilege escalation events (new admin rights, role changes, sudo usage) with subsequent AI service access. High-confidence indicator of insider threat or compromised account.
70 pts + notable
AI-RISK-005
Multi-Vector AI Risk
Fires when a single user triggers detections from 3 or more distinct detection categories within 24 hours. Breadth of activity suggests systematic misuse or advanced threat.
55 pts + notable

Operations

Deployment Checklist

Ten steps to go from zero to operational detection coverage.

  1. Deploy hdsi_ai_rba_common package (macros + transforms)
  2. Deploy detection packages (hdsi_ai_* and hdsi_rba_ai_*)
  3. Install all 9 CSV lookups in app context
  4. Validate CIM data model mappings
  5. Accelerate Web and Endpoint data models (30d+)
  6. Populate ai_sanctioned_entities.csv allowlist
  7. Set ai_detection_config.csv thresholds for your environment
  8. Review ai_department_sensitivity.csv risk multipliers
  9. Enable detections and observe for 1-2 weeks
  10. Tune thresholds using TUNING_GUIDE.md calibration queries
